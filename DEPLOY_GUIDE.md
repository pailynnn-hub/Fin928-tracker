# คู่มือ Deploy FIN Infinity 928 Activity Tracker
ใช้เวลาประมาณ 20-30 นาที ไม่ต้องเขียนโค้ดเพิ่มเลย ทำตามลำดับนี้ครับ

> **อัปเดตเวอร์ชันล่าสุด:** ถ้าเป็นการติดตั้งครั้งแรก ให้รันไฟล์ `supabase/schema.sql` (มีทุกอย่างครบแล้ว รวมกิจกรรมบริการด้วย) — ถ้าเคย deploy ไปแล้ว:
> - เคยรันแค่ schema.sql เวอร์ชันแรก → รัน `migration_v2.sql` แล้วตามด้วย `migration_v3.sql`
> - เคยรัน migration_v2.sql ไปแล้ว → รัน `migration_v3.sql` อย่างเดียว (เพิ่มกิจกรรมบริการ: เยี่ยมลูกค้าหลังขาย/เคลม, CRM, ขอต่อรายชื่อ)
> **ห้ามรัน schema.sql ซ้ำถ้าเคยรันไปแล้วครั้งหนึ่ง** จะ error เพราะตารางซ้ำ

---

## ขั้นตอนที่ 1: สร้างฐานข้อมูล (Supabase) — ฟรี

1. ไปที่ https://supabase.com → กด **Start your project** → สมัครด้วย GitHub หรืออีเมล
2. กด **New Project** → ตั้งชื่อ เช่น `fin928-tracker` → ตั้งรหัสผ่าน database (เก็บไว้ดี ๆ) → เลือก Region ใกล้ไทย (เช่น Singapore) → กด **Create new project** (รอ 1-2 นาที)
3. เมื่อโปรเจกต์พร้อมแล้ว ไปที่เมนูซ้าย **SQL Editor** → กด **New query**
4. เปิดไฟล์ `supabase/schema.sql` ที่แนบมา คัดลอกทั้งหมด วางในช่อง แล้วกด **Run**
   - ถ้าสำเร็จจะเห็นข้อความ "Success. No rows returned"
5. ไปที่เมนูซ้าย **Project Settings** (รูปเฟือง) → **API** → คัดลอก 2 ค่านี้เก็บไว้:
   - **Project URL** (ขึ้นต้นด้วย https://...supabase.co)
   - **anon public key** (เป็นข้อความยาว ๆ)

**เปิดระบบ Email Auth (เปิดอยู่แล้วโดย default):**
ไปที่ **Authentication** → **Providers** → ตรวจสอบว่า Email เปิดอยู่ ถ้าต้องการให้สมัครแล้วใช้ได้ทันทีโดยไม่ต้องกดยืนยันอีเมล ให้ไปที่ **Authentication** → **Settings** → ปิด "Confirm email"

---

## ขั้นตอนที่ 2: อัปโหลดโค้ดขึ้น GitHub

1. ไปที่ https://github.com → สร้างบัญชี (ถ้ายังไม่มี) → กด **New repository** → ตั้งชื่อ `fin928-tracker` → เลือก Private → กด **Create repository**
2. อัปโหลดไฟล์ทั้งหมดในโฟลเดอร์ที่ได้รับ (ลากไฟล์ทั้งหมดใส่ในหน้าเว็บ GitHub ที่ปุ่ม "uploading an existing file" ได้เลย ไม่ต้องใช้ command line)
   - **สำคัญ:** อย่าอัปโหลดไฟล์ `.env.local` (ถ้ามี) เพราะมีข้อมูลลับ — อัปโหลดแค่ `.env.local.example` พอ

---

## ขั้นตอนที่ 3: Deploy ขึ้น Vercel — ฟรี

1. ไปที่ https://vercel.com → กด **Sign up** → เลือก **Continue with GitHub** (เชื่อมบัญชี GitHub)
2. กด **Add New Project** → เลือก repository `fin928-tracker` ที่เพิ่งอัปโหลด → กด **Import**
3. ก่อนกด Deploy ให้เปิดส่วน **Environment Variables** แล้วใส่ 2 ค่าที่คัดลอกไว้จาก Supabase:
   - `NEXT_PUBLIC_SUPABASE_URL` = Project URL
   - `NEXT_PUBLIC_SUPABASE_ANON_KEY` = anon public key
4. กด **Deploy** → รอ 2-3 นาที
5. เสร็จแล้วจะได้ลิงก์เว็บ เช่น `https://fin928-tracker.vercel.app` — นี่คือเว็บแอปของคุณ พร้อมใช้งานจริง แชร์ลิงก์นี้ให้ตัวแทนทุกคนได้เลย

---

## ขั้นตอนที่ 4: ตั้งค่าหัวหน้าทีม (คุณ)

1. เปิดลิงก์เว็บแอป → กด **สมัครสมาชิก** → กรอกชื่อ/อีเมล/รหัสผ่าน → สมัคร
2. กลับไปที่ Supabase → เมนูซ้าย **Table Editor** → เลือกตาราง `profiles`
3. หาแถวที่เป็นอีเมลของคุณ → แก้ค่าคอลัมน์ `role` จาก `agent` เป็น `leader` → กด save
4. กลับไปที่เว็บแอป → ออกจากระบบแล้ว login ใหม่ → ตอนนี้จะเข้าหน้า "ภาพรวมทีม" อัตโนมัติ

---

## ขั้นตอนที่ 5: ให้ตัวแทน 12 คนสมัครใช้งาน

ส่งลิงก์เว็บแอปให้ตัวแทนแต่ละคน ให้กด **สมัครสมาชิก** ด้วยตัวเอง (role จะเป็น `agent` โดยอัตโนมัติ ไม่ต้องตั้งอะไรเพิ่ม)

---

## ขั้นตอนที่ 6: ตั้งเป้าหมาย

Login ด้วยบัญชีหัวหน้าทีม → เมนู **ตั้งเป้าหมาย** → เลือกแท็บ "เป้าหมายกิจกรรมขาย" หรือ "เป้าหมาย Recruit" → เลือกช่วงเวลา (วัน/สัปดาห์/เดือน/ไตรมาส/6เดือน/ปี) → เลือก "ทีมรวม" หรือรายบุคคล → กรอกตัวเลขเป้าหมาย → บันทึก

---

## ฟีเจอร์ใหม่: แคมเปญ/การแข่งขันพิเศษ (มีวันหมดเขต + นับถอยหลัง)

ใช้สำหรับการแข่งขันที่ประกาศเป็นครั้ง ๆ (เช่น แคมเปญพิเศษประจำเดือน) ที่มีเป้า FYP/FYC/จำนวนราย (Life) และวันหมดเขตของตัวเอง แยกจากเป้าหมายปกติ

1. Login ด้วยบัญชีหัวหน้าทีม → เมนู **แคมเปญ** → กรอกชื่อแคมเปญ, เลือกว่าใช้กับทั้งทีมหรือรายบุคคล, วันเริ่ม-วันหมดเขต, ตั้งเป้า FYP/FYC/Life → บันทึก
2. ตัวแทนและหัวหน้าทีมจะเห็นการ์ดแคมเปญที่ด้านบนของหน้ากิจกรรม พร้อม **จำนวนวันที่เหลือแบบนับถอยหลัง** และ **ยอดที่ยังขาดอยู่** (คำนวณใหม่ทุกครั้งที่เปิดหน้าเว็บ อัปเดตตามข้อมูลล่าสุดเสมอ)
3. เมื่อหมดเขต การ์ดจะแสดง "หมดเขตแล้ว" อัตโนมัติ

---

## ฟีเจอร์ใหม่: Recruiting Challenge (การแข่งขันชวนคนเข้าทีม)

ตัวแทนและ**หัวหน้าทีมก็แข่งได้ด้วย** — ทุกคนกรอกกิจกรรมชวนคนของตัวเอง (ตั้งคำถามเกณฑ์, ชวน/ขายอาชีพ, ส่งสัมภาษณ์, สอบผ่าน, ออกโค้ด, Real Agent, FAS, FAP)

1. เมนู **Recruit** — ทุกคน (agent และ leader) เห็นเมนูนี้
2. กรอกกิจกรรมวันนี้ของตัวเอง → ดูความคืบหน้าเทียบเป้าหมายส่วนตัว
3. ด้านล่างมี **กระดานอันดับ** ให้เห็นทุกคนในทีม (รวมหัวหน้าทีม) เรียงตามจำนวนที่ได้ออกโค้ด

---

## ฟีเจอร์ใหม่: หัวหน้าทีมกรอก/แก้ไขข้อมูลแทนตัวแทนได้

จากหน้า **ภาพรวมทีม** → กดปุ่ม **"แก้ไขข้อมูล"** ที่แถวของตัวแทนคนนั้น → เลือกวันที่ที่ต้องการแก้ → สลับแท็บ "กิจกรรมขาย" หรือ "กิจกรรม Recruit" → แก้ตัวเลขแล้วบันทึกได้ทันที (เหมือนกรอกแทนตัวแทนคนนั้นเอง)

---
